# docs.rs: building fewer targets by default

<h1 id="building-fewer-targets-by-default"><a class="anchor" href="#building-fewer-targets-by-default" aria-hidden="true"></a>
Building fewer targets by default</h1>
<p>On <strong>2026-05-01</strong>, docs.rs will make a <strong>breaking</strong> change to its build
behavior.</p>
<p>Today, if a crate does not define a <code>targets</code> list in its
<a rel="external" href="https://docs.rs/about/metadata">docs.rs metadata</a>, docs.rs builds documentation for a default
list of five targets.</p>
<p>Starting on <strong>2026-05-01</strong>, docs.rs will instead build documentation for only
the default target unless additional targets are requested explicitly.</p>
<p>This is the next step in a change we first introduced in 2020, when docs.rs
added support for opting into fewer build targets. Most crates do not compile
different code for different targets, so building fewer targets by default is a
better fit for most releases. It also reduces build times and saves resources on
docs.rs.</p>
<p>This change only affects:</p>
<ol>
<li>new releases</li>
<li>rebuilds of old releases</li>
</ol>
<h2 id="how-is-the-default-target-chosen"><a class="anchor" href="#how-is-the-default-target-chosen" aria-hidden="true"></a>
How is the default target chosen?</h2>
<p>If you do not set <code>default-target</code>, docs.rs uses the target of its build
servers: <code>x86_64-unknown-linux-gnu</code>.</p>
<p>You can override that by setting <code>default-target</code> in your
<a rel="external" href="https://docs.rs/about/metadata">docs.rs metadata</a>:</p>
<pre class="giallo z-code"><code data-lang="toml"><span class="giallo-l"><span>[</span><span>package</span><span>.</span><span>metadata</span><span>.</span><span>docs</span><span>.</span><span>rs</span><span>]</span></span>
<span class="giallo-l"><span class="z-variable">default-target</span><span> =</span><span class="z-punctuation z-definition z-string z-string"> &quot;</span><span class="z-string z-quoted z-string">x86_64-apple-darwin</span><span class="z-punctuation z-definition z-string z-string">&quot;</span></span></code></pre><h2 id="how-do-i-build-documentation-for-additional-targets"><a class="anchor" href="#how-do-i-build-documentation-for-additional-targets" aria-hidden="true"></a>
How do I build documentation for additional targets?</h2>
<p>If your crate needs documentation to be built for more than the default target,
define the full list explicitly in your <code>Cargo.toml</code>:</p>
<pre class="giallo z-code"><code data-lang="toml"><span class="giallo-l"><span>[</span><span>package</span><span>.</span><span>metadata</span><span>.</span><span>docs</span><span>.</span><span>rs</span><span>]</span></span>
<span class="giallo-l"><span class="z-variable">targets</span><span> =</span><span class="z-punctuation z-definition z-array"> [</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-string z-string">    &quot;</span><span class="z-string z-quoted z-string">x86_64-unknown-linux-gnu</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-meta z-array">,</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-string z-string">    &quot;</span><span class="z-string z-quoted z-string">x86_64-apple-darwin</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-meta z-array">,</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-string z-string">    &quot;</span><span class="z-string z-quoted z-string">x86_64-pc-windows-msvc</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-meta z-array">,</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-string z-string">    &quot;</span><span class="z-string z-quoted z-string">i686-unknown-linux-gnu</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-meta z-array">,</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-string z-string">    &quot;</span><span class="z-string z-quoted z-string">i686-pc-windows-msvc</span><span class="z-punctuation z-definition z-string z-string">&quot;</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-array">]</span></span></code></pre>
<p>When <code>targets</code> is set, docs.rs will build documentation for exactly those
targets.</p>
<p>docs.rs still supports any target available in the Rust toolchain. Only the
default behavior is changing.</p>
