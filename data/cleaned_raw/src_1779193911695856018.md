# Announcing Rust 1.94.1

<p>The Rust team has published a new point release of Rust, 1.94.1. Rust is a
programming language that is empowering everyone to build reliable and
efficient software.</p>
<p>If you have a previous version of Rust installed via rustup, getting Rust 1.94.1 is as easy as:</p>
<pre class="giallo z-code"><code data-lang="plain"><span class="giallo-l"><span>rustup update stable</span></span></code></pre>
<p>If you don't have it already, you can <a rel="external" href="https://www.rust-lang.org/install.html">get <code>rustup</code></a> from the appropriate page on our website.</p>
<h2 id="what-s-in-1-94-1"><a class="anchor" href="#what-s-in-1-94-1" aria-hidden="true"></a>
What's in 1.94.1</h2>
<p>Rust 1.94.1 resolves three regressions that were introduced in the 1.94.0 release.</p>
<ul>
<li><a rel="external" href="https://github.com/rust-lang/rust/pull/153634">Fix <code>std::thread::spawn</code> on wasm32-wasip1-threads</a></li>
<li><a rel="external" href="https://github.com/rust-lang/rust/pull/153491">Remove new methods added to <code>std::os::windows::fs::OpenOptionsExt</code></a>
The new methods were unstable, but the trait itself is not sealed and so
cannot be extended with non-default methods.</li>
<li><a rel="external" href="https://github.com/rust-lang/rust-clippy/pull/16685">Clippy: fix ICE in <code>match_same_arms</code></a></li>
<li><a rel="external" href="https://github.com/rust-lang/cargo/pull/16787">Cargo: downgrade curl-sys to 0.4.83</a>
This fixes certificate validation error for some users on some versions of
FreeBSD. See <a rel="external" href="https://github.com/rust-lang/cargo/issues/16357">this issue</a>
for more details.</li>
</ul>
<p>And a security fix:</p>
<ul>
<li><a rel="external" href="https://github.com/rust-lang/cargo/pull/16769">Cargo: Update tar to 0.4.45</a>
This resolves <a rel="external" href="https://www.cve.org/CVERecord?id=CVE-2026-33055">CVE-2026-33055</a> and <a rel="external" href="https://www.cve.org/CVERecord?id=CVE-2026-33056">CVE-2026-33056</a>. Users of crates.io are not affected.
See <a rel="external" href="https://blog.rust-lang.org/2026/03/21/cve-2026-33056/">blog</a> for more details.</li>
</ul>
