# Announcing Rust 1.95.0

<p>The Rust team is happy to announce a new version of Rust, 1.95.0. Rust is a programming language empowering everyone to build reliable and efficient software.</p>
<p>If you have a previous version of Rust installed via <code>rustup</code>, you can get 1.95.0 with:</p>
<pre class="giallo z-code"><code data-lang="shellsession"><span class="giallo-l"><span>$</span><span> rustup update stable</span></span></code></pre>
<p>If you don't have it already, you can <a rel="external" href="https://www.rust-lang.org/install.html">get <code>rustup</code></a> from the appropriate page on our website, and check out the <a rel="external" href="https://doc.rust-lang.org/stable/releases.html#version-1950-2026-04-16">detailed release notes for 1.95.0</a>.</p>
<p>If you'd like to help us out by testing future releases, you might consider updating locally to use the beta channel (<code>rustup default beta</code>) or the nightly channel (<code>rustup default nightly</code>). Please <a rel="external" href="https://github.com/rust-lang/rust/issues/new/choose">report</a> any bugs you might come across!</p>
<h2 id="what-s-in-1-95-0-stable"><a class="anchor" href="#what-s-in-1-95-0-stable" aria-hidden="true"></a>
What's in 1.95.0 stable</h2>
<h3 id="cfg-select"><a class="anchor" href="#cfg-select" aria-hidden="true"></a>
<code>cfg_select!</code></h3>
<p>Rust 1.95 introduces a
<a rel="external" href="https://doc.rust-lang.org/stable/std/macro.cfg_select.html"><code>cfg_select!</code></a>
macro that acts roughly similar to a compile-time <code>match</code> on <code>cfg</code>s. This
fulfills the same purpose as the popular
<a rel="external" href="https://crates.io/crates/cfg-if"><code>cfg-if</code></a> crate, although with a different
syntax. <code>cfg_select!</code> expands to the right-hand side of the first arm whose
configuration predicate evaluates to <code>true</code>. Some examples:</p>
<pre class="giallo z-code"><code data-lang="rust"><span class="giallo-l"><span class="z-entity z-name z-function">cfg_select!</span><span> {</span></span>
<span class="giallo-l"><span class="z-variable">    unix</span><span class="z-keyword z-operator"> =&gt;</span><span> {</span></span>
<span class="giallo-l"><span class="z-keyword">        fn</span><span class="z-entity z-name z-function"> foo</span><span>(</span><span>)</span><span> {</span><span class="z-comment"> /*</span><span class="z-comment"> unix specific functionality </span><span class="z-comment">*/</span><span> }</span></span>
<span class="giallo-l"><span>    }</span></span>
<span class="giallo-l"><span class="z-variable">    target_pointer_width</span><span class="z-keyword z-operator z-assignment z-keyword z-operator"> =</span><span class="z-punctuation z-definition z-string z-string"> &quot;</span><span class="z-string z-quoted z-string">32</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-keyword z-operator"> =&gt;</span><span> {</span></span>
<span class="giallo-l"><span class="z-keyword">        fn</span><span class="z-entity z-name z-function"> foo</span><span>(</span><span>)</span><span> {</span><span class="z-comment"> /*</span><span class="z-comment"> non-unix, 32-bit functionality </span><span class="z-comment">*/</span><span> }</span></span>
<span class="giallo-l"><span>    }</span></span>
<span class="giallo-l"><span class="z-variable">    _</span><span class="z-keyword z-operator"> =&gt;</span><span> {</span></span>
<span class="giallo-l"><span class="z-keyword">        fn</span><span class="z-entity z-name z-function"> foo</span><span>(</span><span>)</span><span> {</span><span class="z-comment"> /*</span><span class="z-comment"> fallback implementation </span><span class="z-comment">*/</span><span> }</span></span>
<span class="giallo-l"><span>    }</span></span>
<span class="giallo-l"><span>}</span></span>
<span class="giallo-l"></span>
<span class="giallo-l"><span class="z-storage z-type">let</span><span class="z-variable"> is_windows_str</span><span class="z-keyword z-operator z-assignment z-keyword z-operator"> =</span><span class="z-entity z-name z-function"> cfg_select!</span><span> {</span></span>
<span class="giallo-l"><span class="z-variable">    windows</span><span class="z-keyword z-operator"> =&gt;</span><span class="z-punctuation z-definition z-string z-string"> &quot;</span><span class="z-string z-quoted z-string">windows</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span>,</span></span>
<span class="giallo-l"><span class="z-variable">    _</span><span class="z-keyword z-operator"> =&gt;</span><span class="z-punctuation z-definition z-string z-string"> &quot;</span><span class="z-string z-quoted z-string">not windows</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span>,</span></span>
<span class="giallo-l"><span>}</span><span>;</span></span></code></pre><h3 id="if-let-guards-in-matches"><a class="anchor" href="#if-let-guards-in-matches" aria-hidden="true"></a>
if-let guards in matches</h3>
<p>Rust 1.88 stabilized <a rel="external" href="https://blog.rust-lang.org/2025/06/26/Rust-1.88.0/#let-chains">let chains</a>. Rust
1.95 brings that capability into match expressions, allowing for conditionals
based on pattern matching.</p>
<pre class="giallo z-code"><code data-lang="rust"><span class="giallo-l"><span class="z-keyword z-control">match</span><span class="z-variable"> value</span><span> {</span></span>
<span class="giallo-l"><span class="z-entity z-name z-type">    Some</span><span>(</span><span class="z-variable">x</span><span>)</span><span class="z-keyword z-control"> if</span><span class="z-storage z-type"> let</span><span class="z-entity z-name z-type"> Ok</span><span>(</span><span class="z-variable">y</span><span>)</span><span class="z-keyword z-operator z-assignment z-keyword z-operator"> =</span><span class="z-entity z-name z-function"> compute</span><span>(</span><span class="z-variable">x</span><span>)</span><span class="z-keyword z-operator"> =&gt;</span><span> {</span></span>
<span class="giallo-l"><span class="z-punctuation z-definition z-comment z-comment">        //</span><span class="z-comment z-line z-double-slash z-comment"> Both `x` and `y` are available here</span></span>
<span class="giallo-l"><span class="z-entity z-name z-function">        println!</span><span>(</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span class="z-string z-quoted z-string">{</span><span class="z-string z-quoted z-string">}</span><span class="z-string z-quoted z-string">, </span><span class="z-string z-quoted z-string">{</span><span class="z-string z-quoted z-string">}</span><span class="z-punctuation z-definition z-string z-string">&quot;</span><span>,</span><span class="z-variable"> x</span><span>,</span><span class="z-variable"> y</span><span>)</span><span>;</span></span>
<span class="giallo-l"><span>    }</span></span>
<span class="giallo-l"><span class="z-variable">    _</span><span class="z-keyword z-operator"> =&gt;</span><span> {</span><span>}</span></span>
<span class="giallo-l"><span>}</span></span></code></pre>
<p>Note that the compiler will not currently consider the patterns matched in <code>if let</code> guards as part of the exhaustiveness evaluation of the overall match, just
like <code>if</code> guards.</p>

<h3 id="destabilized-json-target-specs"><a class="anchor" href="#destabilized-json-target-specs" aria-hidden="true"></a>
Destabilized JSON target specs</h3>
<p>Rust 1.95 removes support on stable for passing a custom target specification
to <code>rustc</code>. This should <strong>not</strong> affect any Rust users using a fully stable
toolchain, as building the standard library (including just <code>core</code>) already
required using nightly-only features.</p>
<p>We're also gathering use cases for custom targets on the <a rel="external" href="https://github.com/rust-lang/rust/issues/151528">tracking issue</a>
as we consider whether some form of this feature should eventually be stabilized.</p>
